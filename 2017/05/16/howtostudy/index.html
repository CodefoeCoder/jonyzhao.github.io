这就是你想知道的学习方法

这两天断断续续看了一位T4专家的分享会的视频，主题是“化知识为智慧”。听下来有几点感触：（1）他对自己所接触到的信息有强烈整理欲望、同时热衷于反思和更新自己学习方法。（2）坚持他这套方法需要消耗常人难以想象的时间，侧面说明他做事的效率和自律性高的吓人。

好了，下面简要介绍一下他的演讲主题，同时也参杂一点自己的体会。

如果你在阅读过程中感到枯燥也很正常，因为这些方法论既高度概括，又像老生常谈。这里插播一个小段子：有人担心现在大家都在学编程了，各种培训机构在办学习班，网上也有各种开源的库可供学习，是不是程序员马上就要烂大街、不值钱了？某高手指出：大可不必担心。因为编程所需要的逻辑能力和学习能力对绝大部分人来说是难以忍受的枯燥，这就从生物学上保证了合格程序员的相对稀缺性。（开个玩笑哈，这就更说明学习方法的重要性了对不）

学习的四个阶段

整套方法论是建立在完整践行者四个阶段的基础上的。

  • 收集信息
  • 提炼知识
  • 掌握要点
  • 迁移运用

1. 收集信息

这也是我们看到有用信息时下意识的行为：收集。

  • 阶段特点：碎片化
  • 阶段表现：转载（分享）、收藏、点赞

  • 方法优化：

    • 推荐将不同平台上看到的文章（知乎、微信、新闻客户端等）统一收集到一个平台上。那位专家推荐的是evernote，基于以下优点：

      • 按目录、按标签进行二维分类整理
        只有目录这个维度的话会造成归类困扰，实践一下你就知道。困扰来自于同一篇文章你可能想同时放到两个目录下，以及目录树层级爆炸。如果暂时没有时间，那么，我的习惯是放到一个temp的目录下。

      • 文章可以二次编辑，有助于重新排版加批注

        不要惊讶，他中意的文章会自己重新排版，在这过程中理清思路。我从小学就这么干，没电脑时整理成剪报，后来整理成word。但再后来在互联网上接触的信息多了，实在没坚持下来。不过对于evernote，个人持保留意见。其大火的时候我使用过一年多，后来坚定地转向了onenote。知乎上我回答过这个问题：[Evernote 有哪些缺点？](https://www.zhihu.com/question/20450025/answer/52713285) onenote没有原生支持加标签，但使用过其强大搜索功能后，标签的意义也没那么大了。

    • 定期删除丧失时效性、价值不高的文章
      真正做到这一点是需要忍痛割爱的。也是强迫自己消化知识的方法。我的做法是不删除，而是放入一个叫outdated的目录下。

2. 提炼知识

收藏的文章被翻出来重读并加以精简提炼，才能从信息变成知识。

  • 阶段特点：单点结构
  • 阶段表现：符合艾宾浩斯遗忘曲线

  • 方法优化：对于需要精读的文章，那是要写评论做笔记的。而且为了防止文章观点偏颇，应当找同类的多篇文章交叉验证，可以尝试写“行业报告”。

    关于写行业报告，我的第一反应就是这要求太高了吧！不过他说，不必拘泥于标准的格式。尽量像模像样地写，然后你再读专业机构发布的行业报告，会发现很多时候他们只是数据更全，观点其实不咋的。总之，90%的人停留在提炼知识这一步。

3. 掌握核心

你提炼出的知识，只能表明你对其持认同态度，并不能说明你掌握了它。

  • 阶段特点：形成体系
  • 阶段表现：复述观点，能与人讨论
  • 方法优化：这一阶段的核心是输出。以什么样的方式输出取决于个人偏好，例如小组分享会之类的。
    关于分享交流，本人暂时停留在写博客答知乎上，而且大多不是跨专业地去认识一个事物。

4. 迁移运用

知识本身不是力量。你拥有的知识再多，也不一定表明你很有智慧。这也是“读书无用论”这种错误观点在某种程度上流行的原因之一。化知识为智慧的重要指标，就是你是否能做到迁移运用。

  • 阶段特点：有力武器
  • 阶段表现：用你的方法论处理新的问题
  • 方法总结（以分析行业发展为例）：
    • 点：形成的一个个分析工具
    • 线：运用多个分析工具的一个熟练流程，也就是分析一个问题的方法论
    • 面：通过多个问题的分析，得到行业趋势
    • 体：多个行业的认知形成自己的认知体系
    • 流形？结合当下和历史的进行，形成自己的判断
      本人没有尝试过进行如此规模的总结和运用，而且没有想过结合时间维度进行再优化，佩服。

好了，本篇就说到这儿，是不是觉得没什么神秘的。写出来的主要目的是告诉大家也告诉自己：这套方法是行之有效的。如果和你的想法不谋而合（产生了我果然是个天才咩嘿嘿这种情绪），那么请坚定地继续贯彻下去。当然，如果有自己独到的方法，也欢迎交流。
