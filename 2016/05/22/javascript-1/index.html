JS是啥，能干啥，怎么干

之前我对javascript的理解是它和java其实没啥关系，仅此而已。现在才知道js对web开发是非常重要的东西。那么js到底是什么？

  • 首先，js是运行在客户端的一种脚本语言（区别于编译型语言），目前所有浏览器都内置了js解释器，js在浏览器的执行过程点这里 ；

  • 其次，js是弱类型的、基于事件和对象的；

  • 最后，js是github上的top language（据说Node.js起到了推波助澜的作用），可见其活跃程度之大。

js能干什么，一言以哔之：

null vs undefined

null和undefined有区别。

使用undefined的情况：

  • 调用函数func(a,b)时少传了一个参数b写成func(a)，则b为undefined.
  • 访问了obj的不存在的属性如obj.whoknows，则返回undefined.

使用null的情况：

  • DOM的API里有时将空的引用设置为null。例如getElementById失败时将返回一个null.

函数

一等公民

五月里有人这样说：
Functions are first class objects in javascript.

（这里用了小说《东京塔》的梗，有人看出来了吗）

言归正传。什么叫first-class obj？其实就是说函数和其他对象一样被对待，可以被动态创建、保存、作为参数传递。更具体地说，我找到了个比较好的说法：

(1) A function is an instance of the Object type.
(2) A function can have properties and has a link back to its constructor method.
(3) You can store the function in a variable.
(4) You can pass the function as a parameter to another function.
(5) You can return the function from a function.

正是赋予函数这种地位，使得js非常灵活和动态。
例如，根据上述第(4)条，可以将框架逻辑和具体作用剥离。想象我要从一个列表中根据某中要求筛选出符合条件的组成新的列表。具体什么要求可以另写一个函数作为filter（好怀念数字信号处理中的filter啊），而该函数只负责框架逻辑：遍历筛选、新建。这样在需求增加时，只需改动filter，从而节省代码量，架构也更优美。延伸阅读：How javascript objects work?
